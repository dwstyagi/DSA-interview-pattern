# Pattern 11b — Graph Fundamentals (Read Before Graph DFS/BFS/Algorithms)

## Why This File Exists

Patterns 11 (BFS), 12 (DFS), 15 (Union-Find), and 20 (Graph Algorithms)
all assume you're comfortable with graph representation and traversal.

If you jump straight to "Number of Islands" or "Course Schedule" without
understanding how graphs are built and traversed from scratch, the code
feels like magic you're memorizing rather than logic you're deriving.

This file covers the **building blocks** — representation, traversal,
and the 5-6 fundamental problems that every graph question reduces to.

---

## How a Graph Differs from a Tree

```
Tree:
  - exactly one root
  - no cycles
  - exactly one path between any two nodes
  - n nodes → n-1 edges

Graph:
  - any node can be a starting point
  - may have cycles  → need a VISITED set
  - may have multiple paths between nodes
  - may be disconnected → need to try ALL unvisited nodes as start
  - n nodes → anywhere from 0 to n*(n-1) edges

A tree is just a special case of a graph.
Everything you learned in 11a carries over — just add visited tracking.
```

## Representations

### 1. Adjacency List (use this by default)

```ruby
# undirected graph
graph = Hash.new { |h, k| h[k] = [] }
edges.each do |u, v|
  graph[u] << v
  graph[v] << u
end

# directed graph (only one direction)
edges.each do |u, v|
  graph[u] << v
end

# weighted graph
edges.each do |u, v, w|
  graph[u] << [v, w]
  graph[v] << [u, w]
end
```

**Space:** O(V + E). Good for sparse graphs (most interview problems).

---

### 2. Adjacency Matrix

```ruby
matrix = Array.new(n) { Array.new(n, 0) }
edges.each do |u, v|
  matrix[u][v] = 1
  matrix[v][u] = 1    # undirected
end
```

**Space:** O(V²). Use only when the graph is dense or given as a matrix input.

---

### 3. Implicit Graph (grid or state space)

```ruby
# no explicit edge list — neighbors computed on the fly
dirs = [[1,0],[-1,0],[0,1],[0,-1]]
neighbors = ->(r, c) {
  dirs.filter_map do |dr, dc|
    nr, nc = r + dr, c + dc
    [nr, nc] if nr.between?(0, rows-1) && nc.between?(0, cols-1)
  end
}
```

Most grid problems (Number of Islands, Rotten Oranges) use this form.

## Core Templates

### 1. DFS on Graph (iterative — no stack overflow risk)

```ruby
def dfs(graph, start)
  visited = {}
  stack = [start]

  while (node = stack.pop)
    next if visited[node]
    visited[node] = true

    # process node here
    puts node

    graph[node].each { |nei| stack << nei unless visited[nei] }
  end
end
```

---

### 2. DFS on Graph (recursive)

```ruby
def dfs_recursive(graph, node, visited = {})
  return if visited[node]
  visited[node] = true

  # process node here
  graph[node].each { |nei| dfs_recursive(graph, nei, visited) }
end
```

---

### 3. BFS on Graph

```ruby
def bfs(graph, start)
  visited = { start => true }
  queue = [start]
  distance = { start => 0 }

  until queue.empty?
    node = queue.shift
    # process node here

    graph[node].each do |nei|
      unless visited[nei]
        visited[nei] = true
        distance[nei] = distance[node] + 1
        queue << nei
      end
    end
  end

  distance
end
```

---

### 4. Count Connected Components (the most fundamental graph problem)

```ruby
def count_components(n, edges)
  graph = Hash.new { |h, k| h[k] = [] }
  edges.each { |u, v| graph[u] << v; graph[v] << u }

  visited = {}
  components = 0

  (0...n).each do |node|
    next if visited[node]
    # unvisited node = new component
    dfs_recursive(graph, node, visited)
    components += 1
  end

  components
end
```

**This pattern — "iterate all nodes, DFS/BFS from unvisited ones, count" —
is the skeleton of at least 15 graph problems.**

---

### 5. Grid as Graph (Number of Islands template)

```ruby
def num_islands(grid)
  rows, cols = grid.length, grid[0].length
  count = 0

  dfs = ->(r, c) {
    return if r < 0 || r >= rows || c < 0 || c >= cols
    return if grid[r][c] != '1'
    grid[r][c] = '0'          # mark visited in-place
    dfs.call(r+1, c); dfs.call(r-1, c)
    dfs.call(r, c+1); dfs.call(r, c-1)
  }

  (0...rows).each do |r|
    (0...cols).each do |c|
      if grid[r][c] == '1'
        dfs.call(r, c)
        count += 1
      end
    end
  end

  count
end
```

### 6. Cycle Detection — Undirected Graph

```ruby
def has_cycle_undirected?(graph, n)
  visited = {}

  dfs = ->(node, parent) {
    visited[node] = true
    graph[node].each do |nei|
      next if nei == parent        # ignore the edge we came from
      return true if visited[nei]  # already visited via different path = cycle
      return true if dfs.call(nei, node)
    end
    false
  }

  (0...n).each do |node|
    return true if !visited[node] && dfs.call(node, -1)
  end

  false
end
```

---

### 7. Cycle Detection — Directed Graph (3-color DFS)

```ruby
def has_cycle_directed?(graph, n)
  # 0 = unvisited, 1 = in current path (gray), 2 = fully done (black)
  color = Array.new(n, 0)

  dfs = ->(node) {
    color[node] = 1                      # mark as in-progress
    graph[node].each do |nei|
      return true if color[nei] == 1     # back edge = cycle
      return true if color[nei] == 0 && dfs.call(nei)
    end
    color[node] = 2                      # mark as complete
    false
  }

  (0...n).each { |node| return true if color[node] == 0 && dfs.call(node) }
  false
end
```

**The distinction matters:**

- Undirected: skip the parent edge, any other visited neighbor = cycle
- Directed: gray (in-stack) neighbor = cycle; black (done) neighbor is safe

### 8. Topological Sort (Kahn's BFS)

```ruby
def topo_sort(n, edges)
  graph  = Hash.new { |h, k| h[k] = [] }
  indegree = Array.new(n, 0)

  edges.each do |u, v|
    graph[u] << v
    indegree[v] += 1
  end

  queue = (0...n).select { |i| indegree[i] == 0 }
  order = []

  until queue.empty?
    u = queue.shift
    order << u
    graph[u].each do |v|
      indegree[v] -= 1
      queue << v if indegree[v] == 0
    end
  end

  order.length == n ? order : []    # empty = cycle detected
end
```

## Key Tricks to Remember

**Always try ALL nodes as starting points**

```ruby
# graphs can be disconnected — one DFS/BFS from node 0 won't reach everything
(0...n).each do |node|
  next if visited[node]
  dfs(node)          # start a new traversal for each unvisited component
end
```

**Mark visited BEFORE enqueuing (BFS) or at entry (DFS)**

```ruby
# BFS: mark when you add to queue, not when you dequeue
# prevents the same node being added to the queue multiple times
visited[nei] = true
queue << nei          # mark BEFORE push, not after pop
```

**In-place visited marking for grids**

```ruby
grid[r][c] = '0'      # overwrite with a sentinel
# saves O(R*C) visited hash — but only when input mutation is allowed
```

**Undirected vs directed cycle detection**

```ruby
# undirected: skip parent edge (nei == parent), any other visited = cycle
# directed:   gray (color == 1) neighbor = cycle; black (color == 2) = safe
# mixing these up is the most common cycle-detection bug
```

**Bipartite = 2-colorable = no odd-length cycle**

```ruby
# BFS: color start node 0, alternate 1/0 for neighbors
# if neighbor already colored same as current → not bipartite
color = Array.new(n, -1)
color[start] = 0
queue.each { |u| graph[u].each { |v|
  if color[v] == -1
    color[v] = 1 - color[u]; queue << v
  elsif color[v] == color[u]
    return false   # same color = odd cycle
  end
}}
```

**Topological sort only works on DAGs**

```ruby
# Kahn's: if final order.length < n → cycle exists (some nodes never reached indegree 0)
# Course Schedule uses this: return false if order.length != n
```

**Dijkstra stale entry guard**

```ruby
d, u = pq.pop
next if d > dist[u]    # stale — we already found a shorter path to u
# without this guard, Dijkstra processes nodes multiple times and gives wrong answers
```

**Multi-source BFS — seed ALL sources at distance 0**

```ruby
# Rotting Oranges, 01 Matrix, Walls and Gates
# DON'T BFS from each source separately (O(V * (V+E)))
# seed ALL sources into queue at the start, single BFS pass → O(V+E)
sources.each { |r, c| queue << [r, c, 0]; visited[[r,c]] = true }
```

## Common Mistakes

**Mistake 1 — forgetting visited set → infinite loop on cycles**

```ruby
# always initialize visited before DFS/BFS
# for undirected graphs, even without explicit cycles,
# A-B edge means B is A's neighbor AND A is B's neighbor → stack overflow
```

**Mistake 2 — BFS for shortest path, DFS for reachability**

```ruby
# DFS finds A path, not necessarily THE SHORTEST path
# if the problem asks "minimum steps / minimum cost" → BFS or Dijkstra
# if it asks "is there a path / count paths / all paths" → DFS is fine
```

**Mistake 3 — using undirected cycle detection on a directed graph**

```ruby
# in directed graphs, A→B and B→A are two separate edges
# the "skip parent" trick doesn't work — you need 3-color DFS
```

**Mistake 4 — Dijkstra on negative weights**

```ruby
# Dijkstra assumes once a node is popped from the heap, its distance is final
# negative edges can invalidate this → use Bellman-Ford instead
```

## Study Order

```
Step 1 — Representation:   build adj list from edges, from matrix, from grid
Step 2 — Basic traversal:  DFS and BFS from a single source (problems 1-4)
Step 3 — Components:       count + identify connected components (problems 5-7)
Step 4 — Grid problems:    implicit graph, 4-directional (problems 8-13)
Step 5 — Directed graphs:  cycle detection, topological sort (problems 14-15)
Step 6 — Advanced BFS/DFS: bipartite, multi-source, path enum (problems 16-22)
Step 7 — Weighted graphs:  Dijkstra, Bellman-Ford (problems 23-28)

After Step 4, you can attempt graph problems in 11_bfs.md and 12_dfs.md.
After Step 7, you're ready for 20_graph_algorithms.md.
```
